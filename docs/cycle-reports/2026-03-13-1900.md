### Meme Sell Root Cause Analysis (this cycle)

`MemeOrchestrator._execute_signal` for SELL delegated to `SimpleExecutor`, which calls
`balance.get(base_asset, 0)`. On Binance testnet/simulation, meme coin base balances
are not properly credited after a mock buy — `balance.get("PEPE", 0)` returns 0, causing
the executor to reject the sell with "No {base_asset} to sell".

The orchestrator already tracks exact position amounts in `self._positions[symbol].amount`.
The fix bypasses the executor balance lookup for SELL orders with a tracked position,
calling `exchange.market_sell()` directly with the known amount.

## Phase 4: Implemented

### Fix #1 — Meme sell direct exchange call (bugfix)

**File:** `agents/memetrader/orchestrator.py`

When `signal.action == SELL` and `self._positions[symbol]` exists with `amount > 0`,
the orchestrator now calls `self.exchange.market_sell(pair, amount)` directly instead
of routing through `SimpleExecutor`. Partial sells (size_pct < 1.0) correctly calculate
`tracked_amount * size_pct`. Falls back to executor when no position is tracked.

**Tests:** 4 new unit tests in `tests/unit/test_meme_sell_fix.py` — all pass.
- `test_sell_uses_tracked_position_not_executor`
- `test_partial_sell_uses_size_pct_of_tracked_amount`
- `test_sell_without_tracked_position_falls_back_to_executor`
