## Phase 4: Implement

### Fix 1: `set_entry_price` Signature (commit `7b794de`)

**Root cause:** `phase3._process_pair` calls:
```python
await self.memory.set_entry_price(base, trade.average_price, trade.filled_size_base)
```
But `IMemory.set_entry_price(symbol, price)` only accepted 2 params → `TypeError` → caught silently → entry price never saved → all positions show `entry_price=0` → PnL cannot be calculated for core pairs.

**Fix:**
- `core/interfaces/__init__.py`: added `size: Optional[float] = None` parameter
- `memory/postgres.py`: updated signature to accept optional `size` (stored for interface compat, table doesn't need it)
- No DB schema change needed (entry_prices table stores price only)

**Tests:** 2 new unit tests verify 2-arg and 3-arg call patterns both succeed.

### Fix 2: Regime Propagation to Trade Records (commit `7b794de`)

**Root cause:** `trades` table had no `regime` column. `record_trade()` never saved regime from `MarketIntel`. All trades classified as "unknown" in analytics.

**Fix:**
- `migrations/007_trades_regime.sql`: `ALTER TABLE trades ADD COLUMN IF NOT EXISTS regime VARCHAR(50)` + backfill from `regime_snapshots`
- `migrations/run_migration.py`: added `007_trades_regime.sql` to ordered migration list
- `memory/postgres.py` `record_trade`: extracts `intel.regime.value` → stores as `$27` in INSERT

**Tests:** 2 new unit tests verify regime is passed correctly for trades with/without intel.
