## Root Cause Analysis

**Why did the new build fail on first deploy attempt?**
- Commit `8448d07` (HEAD when cycle started) introduced an inline `from core.config.settings import Stage` inside `_create_orchestrator()`, which Python treats as a local variable assignment, shadowing the module-level import. Any use of `Stage` before that line → `UnboundLocalError`.
- DGM autoresearch identified and fixed this between cycle phases.
