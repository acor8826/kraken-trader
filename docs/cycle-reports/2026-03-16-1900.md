### Errors Found in Cloud Run Logs (Last 24h)

**Error 1 — CRITICAL (startup crash, new revisions fail to start):**
```
UnboundLocalError: cannot access local variable 'Stage' where it is not associated with a value
  File "/app/api/app.py", line 290, in _create_orchestrator
    if settings.stage != Stage.STAGE_3_FULL:
```
Root cause: `_create_orchestrator()` contained a duplicate inline `from core.config.settings import Stage` ~220 lines into the function. Python's scoping rules treat any name with an assignment anywhere in a function as a local variable throughout that function. This caused `Stage` to be an unbound local at line 290 (before the inline import line). **Every new build since the inline import was added would have crashed on startup.** The currently-running revision `00225-wlg` was built from a prior commit before this bug was introduced.

**Error 2 — MEDIUM (seed improver ledger writes fail):**
```
AttributeError: 'NoneType' object has no attribute 'get'
  File "/app/api/app.py", line 1054, in _summarize_dgm_result
    parts.append(f"eval={ev.get('verdict', ev.get('status', '?'))}")
```
Root cause: DGM phase values can be `None` (e.g. `phases["evaluate"] = None`). The function assumed all phase dict values were non-None. Silently broke the seed improver daily ledger write, meaning improvement history was not being recorded.

**Error 3 — LOW (orchestrator cycle abort on DB error):**
```
asyncpg bind_execute error in get_entry_price()
```
Root cause: `get_entry_price()` re-raised exceptions instead of returning `None`. When the `entry_prices` table query fails (e.g. missing table, connection issue), the Phase3 orchestrator cycle was aborting rather than continuing with a null entry price.

**Error 4 — INFO (not a bug, not actionable):**
`HTTPStatusError: 400 Bad Request` on Binance `/api/v3/account` — leftover from the Binance integration that is no longer used (system now uses Kraken). Non-blocking.

## Top 3 Root Causes

1. **Inline import creating local variable scope conflict** — `Stage` re-imported inside `_create_orchestrator()` when already imported at module level; blocked all new deployments from starting.
2. **Missing None guard in DGM result summarizer** — phase dict values can be `None` but code called `.get()` unconditionally, breaking daily ledger writes.
3. **Error propagation in `get_entry_price`** — exceptions raised instead of graceful `None` fallback, causing orchestrator cycles to abort on transient DB errors.
