## Root Cause Analysis (targeted)

### Incident
Meme sells repeatedly failed with `No position` despite meme strategist/sentinel approving SELL exits.

### Evidence
Cloud Run logs show repeated:
- `[MEME] Direct sell rejected for <symbol>: No position`
- `[SIM] No <symbol> to sell`

### Root Cause
`Phase3Orchestrator._get_portfolio_state()` consumed **all** assets from shared `SimulationExchange` balance (including meme holdings), then main sentinel stop-loss/HWM logic could sell meme assets. This drained `exchange._positions[symbol]`, so later meme sells saw zero balance and failed.
