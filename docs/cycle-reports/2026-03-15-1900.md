## Phase 4: Implement

### Fix #1: SQL `gross_losses` computation — `ABS(SUM() FILTER)` not `ABS(SUM()) FILTER`
**File:** `memory/postgres.py`
**Change:** Moved `)` to wrap the FILTER clause inside ABS:
```sql
-- Before (broken):
COALESCE(ABS(SUM(realized_pnl)) FILTER (WHERE action = 'SELL' AND realized_pnl < 0), 0)
-- After (fixed):
COALESCE(ABS(SUM(realized_pnl) FILTER (WHERE action = 'SELL' AND realized_pnl < 0)), 0)
```
**Impact:** Fixes `/performance` endpoint 500. Unblocks the daily profit review chain (6 PM profit context injection into seed improver). The performance monitoring system was completely blind since deployment.

### Fix #2: Fallback stop-loss for legacy positions with null entry_price
**File:** `agents/orchestrator/base.py`
**Change:** Added `elif current_price and current_price > 0:` branch after `if entry_price > 0:` to assign a conservative stop-loss based on current price when entry_price is null.
```python
elif current_price and current_price > 0:

    take_profit_price = None  # unknown original entry
    logger.warning(f"Position {pair} has null entry_price — fallback stop-loss at {stop_loss_price}")
```
**Impact:** AVAX and DOT positions will now get a protective stop-loss on next orchestrator cycle. Prevents runaway losses from legacy positions with no exit discipline.
